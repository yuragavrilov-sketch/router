## Git workflow

- После каждого завершённого набора изменений создавай git commit.
- Используй осмысленные сообщения: `feat:`, `fix:`, `chore:`, `refactor:` и т.д.
- Добавляй только изменённые файлы (не `git add -A` без необходимости).
- Не пушь в remote без явной просьбы.
